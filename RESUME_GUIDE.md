## 🎯 核心描述 (针对不同岗位优化)

### 方案 A：侧重“工程化与架构设计” (适合寻求大厂、高标准开发职位)
> **描述**：主导基于 Vue 3.5 的高性能极客看板系统开发。设计并实现了一套基于 **CSS Variables + MutationObserver** 的零闪烁动态主题引擎。通过自主研发的 **Bento-style 响应式 Grid 架构** 替代了重型三方可视化库，在提升视觉质感的同时，将核心库首屏 JS 耗时降低了约 30%。

### 方案 B：侧重“性能优化与技术深度” (适合底层原理、极致性能岗位)
> **描述**：攻克了大规模数据矩阵在浏览器主线程渲染的阻塞问题。采用 **Web Workers** 建立独立计算层，实现年度活跃矩阵（Heatmap）的离线聚类算法，确保复杂计算时 UI 仍能维持 60fps 响应。深度定制 Vite 构建流水线，利用模块化懒加载与 SVG 资源压缩，大幅优化了 FCP 与 TTI 指标。

## 🛠️ 技术深度硬核亮点 (面试必杀技)

### 1. 离线计算与主线程解耦
*   **技术细节**：不只是简单的异步处理，而是通过 `Worker` 线程处理 364 天维度的动态数据生成（模拟 500w+ 级数据循环判断）。
*   **亮点**：避免了 JS 长时间运行导致的“页面卡顿（Jank）”，是展示对浏览器渲染机制（Event Loop）深刻理解的最佳案例。

### 2. 精准粒度的主题引擎
*   **技术细节**：利用原生 CSS 变量（Design Tokens）配合 `MutationObserver` 监听 HTML 根节点属性。
*   **亮点**：解决了传统主题切换中可能出现的“白屏闪烁”问题，并实现了组件级别的样式隔离。

### 3. 类型驱动的路由治理 (Type-based Routing)
*   **技术细节**：针对项目中“招募”与“分享”两种不同数据模型的共用详情页，设计了一套带有前缀隔离的 ID 映射系统（如 `recruit_1` vs `share_1`）。
*   **亮点**：解决了跨模块数据 ID 碰撞带来的状态错乱问题，展现了严谨的逻辑治理能力。

### 4. 视觉美学与性能的平衡
*   **技术细节**：使用 **Bento Dashboard** 代替 ECharts。
*   **亮点**：通过原生 CSS Grid 实现复杂的可视化布局，不仅减少了包体积，更通过硬件加速（GPU Acceleration）提升了动效的流畅度。

## 🐞 避坑记录 (展示 Bug 解决能力 - STAR 法则)

**情境 (Situation)**：项目重构过程中，由于删除了冗余的 Mock 变量，导致个人中心页面在特定 Tab 切换时发生静默崩溃，导航功能完全失效。
**任务 (Task)**：定位崩溃原因，并确保在大规模重构时系统的健壮度。
**行动 (Action)**：
1.  使用 Vue Devtools 追踪渲染组件树，发现 Template 中仍残留对已删除变量的引用。
2.  通过封装统一的数据接口（Computed Getters）隔离原始数据流。
3.  为关键导航点增加 Z-index 堆叠上下文管理，解决 UI 覆盖层拦截点击事件的问题。
**结果 (Result)**：修复了崩溃问题，并建立了一套基于 Vue 响应式特性的安全降级机制。

## 💬 高级面试 Q&A (深度对线)

**Q1：为什么在 2024 年你选择减少 ECharts 等图表库的使用，转而纯手工实现 UI？**
> **A**：首先是**包体积优化**。ECharts 即使按需加载，静态资源也超过 100kb+，而原生 CSS Grid + Variables 方案几乎不产生额外体积。其次是**交互定制化**。在玻璃拟态（Glassmorphism）和暗黑模式下，三方库的样式覆盖成本极高，原生方案能通过一行 CSS 变量完美适配，性能上也更容易利用 GPU 硬件加速。

**Q2：如果用户数据量增加 100 倍，你的 Web Worker 方案如何保持高效？**
> **A**：我会引入 **Transferable Objects** 来优化内存通信，将计算结果以二进制（ArrayBuffer）而非 JSON 副本的形式传回主线程。同时，可以考虑使用 **IndexDB** 进行计算结果的持久化缓存，避免重复计算。

**Q3：Vue 3.5 的哪些新特性能优化你的项目？**
> **A**：Vue 3.5 的**响应式解构 (Reactive Props Destructure)** 极大地简化了组件间的数据流传递。此外，改进后的 `Teleport` 延迟机制能让我更优雅地管理全局通知（GeekToast）的渲染层级。
